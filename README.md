# EyeTracker

## Обзор

Система отслеживания взгляда через веб-камеру для настольных приложений. Принцип работы:

1. **Детекция лица** — MediaPipe Face Mesh находит 468 лицевых ландмарков
2. **Извлечение патчей глаз** — из видеокадра вырезаются области левого и правого глаза
3. **Извлечение признаков** — каждый глаз масштабируется до 10x6 пикселей, переводится в градации серого и выравнивается по гистограмме (вектор 120D)
4. **Калибровка** — пользователь кликает по точкам на экране, глядя на них; система обучает маппинг: признаки глаз -> координаты на экране
5. **Предсказание взгляда** — взвешенная гребневая регрессия в реальном времени предсказывает координаты взгляда по признакам глаз

## Требования

- Python 3.12+
- Веб-камера
- macOS / Windows

## Сборка

Приложение собирается в нативный исполняемый файл через PyInstaller:

```bash
make install   # установка зависимостей (включая PyInstaller)
make build     # сборка приложения
```

Результат:
- **macOS** — `dist/EyeTracker.app` (подписывается ad-hoc с entitlements для доступа к камере)
- **Windows** — `dist/EyeTracker.exe` (single-file)

На macOS при первом запуске система запросит разрешение на доступ к камере.

## Запуск

### Собранное приложение

- **macOS**: открыть `dist/EyeTracker.app` или `open dist/EyeTracker.app`
- **Windows**: запустить `dist\EyeTracker.exe`

### Из исходников

```bash
make run
```

Или напрямую:

```bash
poetry run eyetracker
```

## Использование

### Авторизация

При запуске открывается экран "Обзор" с формой входа. После авторизации становятся доступны все пункты бокового меню. В локальной версии принимается любой логин и пароль.

### Боковое меню

| Пункт | Описание |
|-------|----------|
| Обзор | Форма входа / дашборд |
| Демо-трекер | Запуск демо-калибровки (fullscreen) |
| Тесты | Библиотека тестов |
| Создать тест | Создание нового теста |
| Настройки | Выбор монитора, пропуск калибровки, маркер взгляда |
| Помощь | FAQ |

### Создание теста

1. "Создать тест" → "Форма"
2. Заполните название, выберите обложку и добавьте изображения
3. Порядок изображений можно менять перетаскиванием
4. "Создать" — тест сохраняется в библиотеку

Незавершённые черновики автоматически сохраняются и предлагаются к восстановлению при следующем запуске.

### Прохождение теста

1. Откройте тест из библиотеки → "Пройти"
2. Экран готовности → "Начать"
3. Калибровка (кликайте по красным точкам, глядя на них); при включённом "Пропустить калибровку" загружаются ранее сохранённые данные
4. Последовательный показ изображений теста с трекингом взгляда
5. Если включён параметр "Показывать маркер взгляда" — во время теста отображается красная точка в текущей позиции взгляда
6. По завершении результат автоматически сохраняется

### Просмотр результатов

1. Откройте тест → "Результаты"
2. Таблица с датой, пользователем и кнопкой "Посмотреть отчет"
3. Детали записи: табы по изображениям (1, 2, 3, ...) — каждый таб показывает тепловую карту взгляда поверх исходного изображения
4. "Выгрузить отчет" → ZIP-архив с `report.json` и папками `image_N/` (оригинал + тепловая карта + метрики)

### Демо-калибровка

Для проверки трекера без теста:

1. "Демо-трекер" в sidebar
2. Кликните по каждой из 9 красных точек 5 раз, глядя на них
3. Смотрите на зелёную точку 5 секунд (измерение точности)
4. Автоматический переход в режим отслеживания (красная точка следует за взглядом)

Нажмите **Esc** для возврата на начальный экран.

### Превью видео

В левом верхнем углу отображается превью с веб-камеры и наложением ландмарков лица. Нажмите **V** чтобы показать/скрыть превью (работает на любом этапе).

### Режим динамической тренировки

В режиме отслеживания нажмите **T** для включения режима тренировки. Кликайте по экрану, глядя на точку клика — модель дообучается на лету.

### Параметры командной строки

```bash
poetry run eyetracker --help

Параметры:
  --camera INDEX       Индекс устройства камеры (по умолчанию: 0)
  -v, --verbose        Подробное логирование
```

Сглаживание Калмана, детекция моргания и взвешенная гребневая регрессия включены по умолчанию.

## Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| `T` | Включить/выключить режим тренировки (в демо-трекере) |
| `R` | Перекалибровка (в демо-трекере) |
| `V` | Показать/скрыть превью видео |
| `Esc` | Назад / Возврат на начальный экран / Выход |

## Разработка

### Первоначальная настройка

```bash
make setup
```

Команда автоматически:
1. Устанавливает Python 3.12 (`brew install python@3.12` на macOS, `winget install` на Windows)
2. Настраивает Poetry-окружение на Python 3.12 (`poetry env use`)
3. Устанавливает все зависимости (`poetry lock && poetry install`)

Требования: [Homebrew](https://brew.sh) на macOS или [winget](https://learn.microsoft.com/windows/package-manager/winget/) на Windows, а также [Poetry](https://python-poetry.org/docs/#installation).

### Установка зависимостей (если Python уже настроен)

```bash
make install
```

### Тестирование

```bash
make test           # или poetry run pytest
make test-verbose   # или poetry run pytest -v
```

109 тестов покрывают: утилиты, регрессию, детекцию моргания, измерение точности, настройки, мониторы, хранилище тестов, валидацию форм, черновики, авторизацию, метрики взгляда, записи результатов, тепловые карты, экспорт отчётов, форматирование времени.

### Все команды Makefile

```bash
make setup          # Установка Python 3.12 + Poetry env + зависимости
make install        # Установка зависимостей (Python уже настроен)
make run            # Запуск из исходников
make test           # Запуск тестов
make test-verbose   # Запуск тестов с подробным выводом
make build          # Сборка нативного приложения
make clean          # Очистка __pycache__, build, dist
make help           # Справка
```
